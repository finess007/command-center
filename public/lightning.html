Lightning Node Setup
Mac mini ‚Ä¢ LND Neutrino mode ‚Ä¢ ~15-20 min
0 of 8 steps 0%
Step 1
Install Go
Go is required to build LND from source. Install via Homebrew.
brew install go
Step 2
Install LND
Lightning Network Daemon ‚Äî the node software. Install from Homebrew or build from source.
brew install lnd # OR from source: git clone https://github.com/lightningnetwork/lnd cd lnd && make install
Step 3
Configure Neutrino (Light Client)
Neutrino mode ‚Äî no need to download the full 600GB blockchain. Syncs in minutes.
mkdir -p ~/.lnd cat > ~/.lnd/lnd.conf << 'EOF' [Application Options] alias=nova-node listen=0.0.0.0:9735 [Bitcoin] bitcoin.active=true bitcoin.mainnet=true bitcoin.node=neutrino [Neutrino] neutrino.connect=btcd-mainnet.lightning.computer neutrino.feeurl=https://nodes.lightning.computer/fees/v1/btc-fee-estimates.json EOF
Step 4
Start LND & Create Wallet
Start the daemon, then create a new wallet. Save the seed phrase safely!
# Terminal 1: Start LND lnd # Terminal 2: Create wallet lncli create # ‚Üí Choose password, save 24-word seed phrase
Step 5
Fund the Wallet
Generate a new address and send 50-100K sats. Confirm with lncli walletbalance.
# Get a new Bitcoin address lncli newaddress p2wkh # Send 50-100K sats from your wallet # Wait for confirmation, then check: lncli walletbalance
Step 6
Open a Channel
Connect to a well-connected node and open a channel for routing payments.
# Connect to a node (e.g., ACINQ) lncli connect 03864ef025fde8fb587d989186ce6a4a186895ee44a926bfc370e2c366597a3f8f@3.33.236.230:9735 # Open channel (amount in sats) lncli openchannel --node_key 03864ef... --local_amt 50000 # Check channels lncli listchannels
Step 7
Create Scoped Macaroon for Nova
A macaroon is a credential token. Create one with limited permissions for Nova (read + invoice only).
# Create read-only + invoice macaroon for Nova lncli bakemacaroon info:read invoices:read invoices:write offchain:read offchain:write --save_to ~/.lnd/nova.macaroon # Verify it works lncli --macaroonpath ~/.lnd/nova.macaroon getinfo
Step 8
Install Lightning Agent Tools
Lightning Labs' open-source toolkit for AI agents ‚Äî includes lnget (L402 payments), MCP integration, and more.
git clone https://github.com/lightninglabs/lightning-agent-tools cd lightning-agent-tools # Follow setup in README

Bitcoin = Post-AI Trust Layer
The core thesis ‚Ä¢ Feb 12, 2026

In a world dominated by AI, logic-based security collapses. AI can find every bug, generate perfect deepfakes, and create infinite zero-cost attacks. Bitcoin is the only protocol that moves security from a logic problem to a physics problem. This makes BTC the foundational trust layer for everything ‚Äî identity, code integrity, content authenticity, communication.

The 6 Protection Layers
1
Economic Walls
Lightning micropayments per attempt. Free attacks become expensive.
  • Every auth attempt costs sats via Lightning
  • Spam email? 1 sat per send. Bots go bankrupt.
  • API calls require micropayment channels. DDoS = economically impossible.
  • Attack cost scales linearly. Defense cost stays flat.
‚ñº
2
Identity = Private Key
No passwords to phish. Cryptographic signatures can't be deepfaked.
  • Your identity IS your private key ‚Äî not username/password
  • No central database of credentials to breach
  • Deepfake your face? Doesn't matter ‚Äî can you sign with the key?
  • Identity moves from "something you know" to "something you possess"
‚ñº
3
Proof of Authenticity
Timestamp content on Bitcoin. Deepfakes can't match the hash.
  • Hash content (photo, video, doc) and anchor to Bitcoin's timechain
  • "This photo existed at block 900,000" ‚Äî verifiable provenance
  • Deepfakes exist but can't produce a matching pre-creation hash
  • Truth becomes timestamped and immutable
‚ñº
4
Code Integrity
Every commit hashed to Bitcoin. Tampered code won't match.
  • Software releases get their hash anchored to Bitcoin
  • Verify hash against blockchain before running code
  • Supply chain attacks (SolarWinds) detectable instantly
  • Trust moves from "trust the developer" to "verify the hash"
‚ñº
5
Multi-sig Critical Systems
Hardware wallets in separate locations. AI can't access physical vaults.
  • Critical infrastructure protected by multi-signature schemes
  • Hardware signing devices in physically separate locations
  • AI can hack software but can't walk into a vault in Zurich
  • Security moves from cyberspace to physical space ‚Äî AI's weakness
‚ñº
6
Lightning as Immune System
Every machine-to-machine interaction costs a micropayment.
  • IoT devices pay each other fractions of sats for data
  • AI agents need economic stake to interact
  • Sybil attacks become prohibitively expensive at scale
  • The internet's free-rider problem gets solved

Core Concepts
‚ö° Lightning Network ‚ñº
A Layer 2 payment protocol on top of Bitcoin. Enables instant, near-zero-fee transactions by opening payment channels between nodes. Payments route through the network without touching the blockchain until channels close. Think of it as Bitcoin's "pocket cash" ‚Äî the blockchain is the vault, Lightning is the wallet you carry around.
üîÑ Atomic Swap (HTLC) ‚ñº
Hash Time-Locked Contract ‚Äî a cryptographic mechanism that makes trustless exchanges possible. Both parties lock funds with a shared secret hash. Either both get paid, or neither does. No custodian, no trust needed. This is how RailsX/Amboss does Lightning-native DEX trading. The "atomic" part means it's all-or-nothing ‚Äî no partial execution.
üç™ Macaroons ‚ñº
LND's credential system ‚Äî like API keys but better. You can bake macaroons with specific permissions (read-only, invoice-only, etc.) and time limits. For Nova's agent setup, we use a scoped macaroon: info:read + invoices:read/write + offchain:read/write. The node owner keeps the admin macaroon; agents get limited ones.
üí≥ L402 (Lightning HTTP 402) ‚ñº
HTTP status code 402 = "Payment Required." L402 is a standard for machine-to-machine payments over the web. When an AI agent hits an API that returns 402, lnget (by Lightning Labs) automatically pays the Lightning invoice and retries the request. This is how the machine-payable web works ‚Äî APIs charge per request, agents pay automatically.
üåø Taproot Assets ‚ñº
A protocol by Lightning Labs for issuing tokens (stablecoins, NFTs, etc.) on Bitcoin and routing them over Lightning. USDT on Lightning is already live via Taproot Assets. This turns Lightning from a "BTC-only" network into a multi-asset payment rail. Agents can pay in USDT, receive in BTC, all over Lightning.
ü™∂ Neutrino ‚ñº
A light client mode for LND. Instead of downloading the entire Bitcoin blockchain (~600GB), Neutrino only fetches block headers and relevant data. Syncs in minutes instead of days. Perfect for personal nodes. Trade-off: slightly less privacy (you ask full nodes for your transactions), but for a dev/agent setup, it's ideal.
üîë Multi-sig (Multi-Signature) ‚ñº
Requires multiple private keys to authorize a transaction (e.g., 2-of-3). Keys can be stored in physically separate locations. This is Layer 5 of the thesis ‚Äî AI can hack any single point of failure, but it can't simultaneously access hardware in Zurich, Singapore, and a home safe. Critical systems (nuclear codes, power grids, big treasuries) will all move to multi-sig.
üè∑Ô∏è Liquid Network (Blockstream) ‚ñº
A Bitcoin sidechain by Blockstream. Enables faster settlement (2 min vs 10 min), confidential transactions, and token issuance (L-BTC, security tokens, stablecoins). $3.27B TVL, #3 RWA chain globally. Different from Lightning: Liquid is for settlement (traders, institutions), Lightning is for payments (consumers, agents).
Security Concepts
üî• Hot Wallet vs Cold Storage ‚ñº
Hot wallet = connected to internet (Lightning node). Convenient but vulnerable. Cold storage = offline (hardware wallet, paper). Secure but slow. Rule: Lightning = pocket cash (small amounts, accept the risk). Cold storage = vault (serious money). Never hold life savings on a Lightning node.
üßÖ Sybil Attack ‚ñº
Creating many fake identities to overwhelm a network. Free on most platforms (create infinite email accounts). On Lightning, each identity needs economic stake (sats). This is why Layer 6 works ‚Äî the internet gets an immune system. Creating 1M fake accounts costs 1M √ó N sats. Spam becomes a financial decision, not a technical one.
